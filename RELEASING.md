# Releasing Goleo to npm

Goleo ships **nine** npm packages:

| Package | What it is |
|---|---|
| `@goleo/bridge` | Frontend bridge library (TypeScript) |
| `create-goleo-app` | Project scaffolder |
| `@goleo/cli` | The `goleo` CLI (thin Node launcher) |
| `@goleo/cli-darwin-arm64` · `-darwin-x64` · `-linux-arm64` · `-linux-x64` · `-win32-arm64` · `-win32-x64` | Prebuilt CLI binaries, one per OS/CPU. Generated by `cli/npm/build-platform-packages.js`; `@goleo/cli` pulls the matching one via `optionalDependencies`. |

Publishing is automated by `.github/workflows/release.yml` on a `v*` tag, using
npm **trusted publishing** (OIDC) — no `NPM_TOKEN` secret, and provenance is
attached automatically.

## One-time setup

1. **Own the `@goleo` scope.** On npmjs.com create an **organization** named
   `goleo` (free for public packages). If the name is taken, rename the scope
   throughout the repo first.

2. **Configure a Trusted Publisher for each package.** On npmjs.com, for every
   package above: **package → Settings → Trusted Publishing → Add**:
   - Provider: **GitHub Actions**
   - Owner: **daforester**
   - Repository: **goleo**
   - Workflow: **release.yml**

   > **Brand-new package names (first release):** npm lets you pre-configure a
   > trusted publisher for a package you own within your scope before it exists.
   > If a first OIDC publish is still rejected because the name is new, publish
   > that one package once manually to create it (`npm publish --access public`
   > from a local build), then the trusted publisher takes over for all future
   > releases. This mainly affects the six `@goleo/cli-*` names on the very first
   > release.

3. **Make the repository public** (required for provenance).

4. Confirm the **LICENSE** holder (currently `Aaron Parker`).

No `NPM_TOKEN` GitHub secret is required with trusted publishing.

## Cutting a release

Bump all workspaces to the same version, commit, tag, push:

```bash
npm version 0.2.0 --workspaces --no-git-tag-version   # bumps bridge, create-goleo-app, cli/npm
git commit -am "release: v0.2.0"
git tag v0.2.0
git push --follow-tags
```

The `release` workflow then:
1. cross-compiles the six platform binaries (`CGO_ENABLED=0`),
2. builds the `@goleo/cli-*` packages (versions + `@goleo/cli` `optionalDependencies`
   are auto-synced by `build-platform-packages.js`),
3. publishes **platform packages → `@goleo/cli` → `@goleo/bridge` →
   `create-goleo-app`** via OIDC, each with provenance.

You only set three versions; the platform packages inherit `@goleo/cli`'s.

## Manual release (fallback)

Needs the Go toolchain and `npm login` (and, without trusted publishing, an npm
account with publish rights):

```bash
npm run build:all      # tsc + bundle Go source + build platform packages
npm run publish:all    # platform packages, then cli, bridge, create-goleo-app
```

## What users run after a release

```bash
npm install -g @goleo/cli        # installs the matching @goleo/cli-<os>-<arch> automatically
# or
npm create goleo-app@latest my-app
```
