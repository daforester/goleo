# Releasing Goleo to npm

Goleo ships **nine** npm packages:

| Package | What it is |
|---|---|
| `@goleo/bridge` | Frontend bridge library (TypeScript) |
| `create-goleo-app` | Project scaffolder |
| `@goleo/cli` | The `goleo` CLI (thin Node launcher) |
| `@goleo/cli-darwin-arm64` · `-darwin-x64` · `-linux-arm64` · `-linux-x64` · `-win32-arm64` · `-win32-x64` | Prebuilt CLI binaries, one per OS/CPU. Generated by `cli/npm/build-platform-packages.js`; `@goleo/cli` pulls the matching one via `optionalDependencies`. |

Steady-state releases are automated (tag → GitHub Actions → npm **trusted
publishing**, no token). But the **very first** publish has a chicken-and-egg —
trusted publishers can only be attached to packages that already exist — so day
zero is a one-time manual bootstrap. Follow the sections in order.

---

## 0. Prerequisites (once)

- **npm account**, logged in: `npm login` → verify `npm whoami`.
- **Own the `@goleo` scope.** On npmjs.com → *Add Organization* → name `goleo`
  (Free / public). This claims `@goleo`.
- **Names are free.** `npm view @goleo/cli` and `npm view create-goleo-app`
  should both return **E404** (available). If either is taken by someone else,
  rename the scope/package throughout the repo before continuing.
- **Go toolchain** installed (day zero cross-compiles the six binaries) + Node 18+.
- Confirm the **LICENSE** holder (currently `Aaron Parker`).

---

## 1. Day zero — the first publish (creates all 9 packages)

Because a trusted publisher can't be configured until a package exists, do one
manual publish. Use a temporary **Automation** token so 2FA/OTP doesn't block the
nine publishes.

```bash
# a) npmjs.com → Access Tokens → Generate New Token → "Automation" → copy it.

# b) Store it locally just for this bootstrap (writes ~/.npmrc):
npm config set //registry.npmjs.org/:_authToken=<TOKEN>

# c) From the repo root — build + publish everything:
npm install
npm run publish:all      # tsc, bundle Go source, cross-compile 6 binaries,
                         # then publish: platform pkgs → cli → bridge → create-app

# d) Remove the token again (and optionally revoke it on npmjs):
npm config delete //registry.npmjs.org/:_authToken
```

Verify: `npm view @goleo/cli version` shows `0.1.0`, and on any machine
`npm install -g @goleo/cli && goleo version` prints `Goleo v0.1.0`.

> Starting version is `0.1.0` (all workspaces). To debut at `1.0.0` instead, run
> `npm version 1.0.0 --workspaces --no-git-tag-version` before step (c).

---

## 2. Enable trusted publishing (once, after day zero)

Now that the packages exist, wire up OIDC so future releases need no token.

1. **Make the GitHub repo public** (required for provenance).
2. For **each** of the nine packages: npmjs.com → *package → Settings → Trusted
   Publishing → Add a trusted publisher*:
   - Provider: **GitHub Actions**
   - Owner: **daforester**
   - Repository: **goleo**
   - Workflow: **release.yml**

No `NPM_TOKEN` GitHub secret is needed — `.github/workflows/release.yml` already
has `id-token: write` and upgrades to `npm@latest` for the OIDC exchange.

---

## 3. Every release after that (tag → automated)

Bump all workspaces to the same version, commit, tag, push:

```bash
npm version 0.2.0 --workspaces --no-git-tag-version   # bumps bridge, create-goleo-app, cli/npm
git commit -am "release: v0.2.0"
git tag v0.2.0
git push --follow-tags
```

The `release` workflow then:
1. cross-compiles the six platform binaries (`CGO_ENABLED=0`),
2. builds the `@goleo/cli-*` packages (their versions + `@goleo/cli`'s
   `optionalDependencies` are auto-synced by `build-platform-packages.js`),
3. publishes **platform packages → `@goleo/cli` → `@goleo/bridge` →
   `create-goleo-app`** via OIDC, each with provenance.

You only set three versions; the platform packages inherit `@goleo/cli`'s.

---

## Manual release (fallback)

Needs the Go toolchain and npm auth (a token in `~/.npmrc`, as in day zero):

```bash
npm run build:all      # tsc + bundle Go source + build platform packages
npm run publish:all    # platform packages, then cli, bridge, create-goleo-app
```

## What users run after a release

```bash
npm install -g @goleo/cli        # installs the matching @goleo/cli-<os>-<arch> automatically
# or
npm create goleo-app@latest my-app
```
